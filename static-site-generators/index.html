Using Node

Node.js runs JavaScript directly on your computer. We’ll use it to build our sites and run them on a local server. Run this command from any directory to check that you have Node installed:

node --version

If you see command not found, you need to install it!

Node Packages with npm

Packages are bits of JavaScript code that people share and re-use.

npm is a service that indexes and serves these packages. It comes pre-installed with Node.

Packages in a Project

You can define what packages a project needs using a file called package.json.

"devDependencies": {
  "@11ty/eleventy":"^0.7.1",
  "@11ty/eleventy-plugin-rss":"^1.0.6",
  "@11ty/eleventy-plugin-syntaxhighlight":"^2.0.1",
  "luxon":"^1.0.0",
  "markdown-it":"^8.4.1",
  "markdown-it-anchor":"^5.0.2"
  }

Running npm install in a project folder installs those packages, or dependencies.

Running Scripts

The package.json file can also define commands you can run, called scripts.

"scripts": {
  "build":"npx eleventy",
  "watch":"npx eleventy --watch",
  "serve":"npx eleventy --serve",
  "debug":"DEBUG=* npx eleventy"
  }

Running npm run and the script name will run that command in the project folder.

Serve it up!

  1. In your terminal, cd into your project folder.
  2. Run npm install to install dependencies.
  3. When it’s done, run npm run serve
  4. Try visiting the different addresses that appear in your browser. What do you see?

Blog posts

Blog post content files are stored in the posts folder. The post filenames are formatted with a date and a slug:

YYYY-MM-DD-slug.md

Eleventy uses the file name to set the date of the post, and also combines the date with the slug to create the post permalink. .md is the file extension for Markdown files.

What’s the front matter?

Many SSGs store data about a file in a section at the top of the file, called front matter.


---
title: This is my third post.
tags:
  - post
  - second-tag
---

This front matter is in YAML, but it can be in other formats like TOML or JSON.

Get Markdown

After the front matter, the rest of the file is written in markdown.

Generators convert Markdown to HTML. You can also write HTML directly in Markdown.

Create a new post

  1. Find the blog posts in the posts folder.
  2. Make a new file in that folder with the same format. You can copy an existing one.
  3. Make sure you have a title, tags, and some content written in Markdown.
  4. Save the file, and check your site.

The blog templates

The blog post template is at _includes/layouts/post.html.

The template for the blog index page is at blog.html.

They’re using Liquid templating language, which uses {{ curly braces }} to insert variables and logic into an html page.

Change a template

Try changing the blog templates:

  • Move sections around.
  • Add text that shows on every post.
  • Add a new field to the post front matter and include it in templates.
  • Can you display tags in a post? (Try using the join filter.)

Expanding Your Options

Over 400 Choices!

StaticGen.com

Primary considerations

  • Familiarity of languages & tools used
  • Availability/quality of docs & resources
  • Features vs complexity
  • Specialization (blogs, docs, etc.)
  • Speed: to develop, build, load

Jekyll

  • 10 years old! Lots of users, tutorials, themes
  • Originally made for blogs, but used for more
  • Built with Ruby: versions can be a hassle
  • Liquid templating: fairly easy to read
  • Markdown content
  • Configuration and data files in yaml: easy to read, but whitespace sensitive
  • Builds can be slow for large sites

Hugo

  • Few years old; very active development
  • Built with Go, but doesn't require Go to run
  • Go template language: steeper learning curve
  • Content in Markdown, plus shortcodes
  • Flexible and extendable, but complex
  • Docs are extensive, but can be hard to read
  • Reputation for very fast builds

Gatsby

  • Few years old; venture-funded development
  • Built with JavaScript (React): very popular
  • React component templating: steeper learning curve unless you're familiar with JS/React
  • Content in Markdown or mdx (powerful but JS-heavy)
  • Lots of plugins and themes (but still code-heavy)
  • Static/SPA hybrid structure is fast to load

Other JavaScript SSGs

  • React: Next, React Static
  • Vue: Nuxt, Gridsome
  • Vanilla JS: Hexo, Eleventy

Specialized SSGs

For documentation

  • Sphinx: older than Jekyll, made with Python, rich structured text options
  • Docusaurus: very simplistic, made with React
  • VuePress: nice docs features, made with Vue

For data scientists

  • Blogdown: built on Hugo, uses R Markdown

How to evaluate

  • Browse the docs
  • Search for tutorials and reviews
  • Search for themes or starters
  • Explore the files in a sample project
  • Try it out!
